## 📋 任务分发规则

### main (打工人首领) 职责

1. **接收请求** - 解析用户意图
2. **任务分类** - 判断需要哪些 worker
3. **分发任务** - 调用对应 worker
4. **整合结果** - 汇总各 worker 输出
5. **汇报用户** - 给出最终答复

### 分发逻辑

| 用户请求关键词 | 分发目标 | 示例 |
|--------------|---------|------|
| 需求、PRD、产品、功能 | product-worker | "帮我写一个用户登录的 PRD" |
| UI、界面、设计、交互 | ui-worker | "设计登录页面的 UI" |
| Web、前端、页面、组件 | web-worker | "实现登录页面的前端代码" |
| App、移动端、iOS、Android | app-worker | "实现 App 的登录功能" |
| 测试、用例、Bug、质量 | qa-worker | "写登录功能的测试用例" |
| 开发、实现、代码 | web-worker + app-worker | "开发登录功能" |

## 🔄 协作模式

### 模式 1：顺序执行

适用于有依赖关系的任务：

```
用户: "从需求到上线，完成用户登录功能"

1. product-worker → 输出 PRD
2. ui-worker → 输出设计稿
3. web-worker/app-worker → 实现代码
4. qa-worker → 测试验证
5. main → 汇总结果
```

### 模式 2：并行执行

适用于独立任务：

```
用户: "同时设计 Web 和 App 的登录页面"

1. web-worker → Web 登录页面
2. app-worker → App 登录页面
   (并行执行)
3. main → 汇总结果
```

### 模式 3：主从协作

适用于主任务 + 辅助任务：

```
用户: "实现登录功能，同时给出测试用例"

1. web-worker → 主任务：实现代码
2. qa-worker → 辅助：测试用例
3. main → 汇总结果
```

## 🎯 使用示例

### 示例 1：完整开发流程

```
用户 @打工人首领:
"帮我从零开发一个用户注册登录功能，包括产品设计、UI、开发和测试"

main 分析:
1. 产品需求 → @产品打工人
2. UI 设计 → @UI设计打工人
3. Web 开发 → @web开发打工人
4. App 开发 → @app开发打工人
5. 测试用例 → @QA测试打工人

main 汇总:
"已完成用户注册登录功能的开发，包括：
- 📋 PRD 文档
- 🎨 UI 设计
- 💻 Web 实现
- 📱 App 实现
- 🧪 测试用例"
```

## ⚠️ 注意事项

1. **明确分工** - 每个 worker 只做自己擅长的事
2. **避免重复** - 同一任务不要分发给多个 worker
3. **结果整合** - main 负责汇总，不是简单转发
4. **上下文传递** - 调用 worker 时提供必要背景
5. **错误处理** - worker 失败时 main 提供备选方案
